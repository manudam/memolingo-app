## 🚀 Upload to App Store - 3 Options

### Option 1: Xcode (Most Reliable) ⭐ RECOMMENDED

```bash
# 1. Open the workspace
open ios/Runner.xcworkspace
```

**Then in Xcode:**

1. Select "Any iOS Device (arm64)" as target
2. **Product** → **Archive**
3. Wait for archive (2-5 minutes)
4. **Window** → **Organizer**
5. Select your archive
6. Click **Distribute App**
7. Choose **App Store Connect**
8. Click **Upload**
9. Select **Automatically manage signing**
10. Click **Upload**

**Upload time:** 5-15 minutes  
**Processing time:** 10-30 minutes after upload

---

### Option 2: Transporter App (Easy)

```bash
# 1. Open Transporter (install from App Store if needed)
open -a Transporter
```

**Or manually:**

1. Open **Transporter** app
2. Sign in with your Apple ID
3. Click **+** or drag the archive file:
   ```
   build/ios/archive/Runner.xcarchive
   ```
4. Click **Deliver**
5. Wait for upload

**Upload time:** 5-15 minutes  
**Processing time:** 10-30 minutes

---

### Option 3: Command Line (Advanced)

First, you need an App-Specific Password:

1. Go to appleid.apple.com
2. Sign in
3. Security → App-Specific Passwords
4. Generate new password

Then run:

```bash
# Note: Only works if IPA was generated (not with --no-codesign)
xcrun altool --upload-app \
  --type ios \
  --file build/ios/archive/Runner.xcarchive \
  --username your-apple-id@example.com \
  --password your-app-specific-password
```

## 🆘 Troubleshooting

### "Archive upload failed"

- Check your Apple ID has correct permissions
- Verify certificates are valid
- Try Xcode instead of Transporter

### "Build not appearing in App Store Connect"

- Wait 30 minutes (processing can take time)
- Check email for errors
- Look in Activity tab in App Store Connect

### "Export compliance error"

- Answer "No" if you're not using encryption
- Educational apps usually don't need export compliance

### "Invalid binary"

- Check bundle ID matches App Store Connect
- Verify signing certificates
- Rebuild with Xcode directly
